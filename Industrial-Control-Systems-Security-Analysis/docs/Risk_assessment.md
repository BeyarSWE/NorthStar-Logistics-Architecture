### **Risk Assessment: Väst Vindpark**

| Risk Object | Description & Threat (STRIDE) | Inherent Risk (L/I/V) | Current Status | Proposed Mitigation (IEC 62443-3-3) | Residual Risk (L/I/V) |
| :--- | :--- | :---: | :--- | :--- | :---: |
| **User Rights (IAM)** | Users have too much access. If someone steals an account, they can change the PLC settings. <br> **Threat:** Elevation of Privilege | 5 / 5 / **25** | Many people use the same accounts and simple passwords. | **SR 2.1:** Give users only the access they need for their job. <br> **SR 1.1:** Use unique accounts for everyone. | 2 / 2 / **4** |
| **Passwords & Login** | There is no multi-factor authentication (MFA). It is easy for hackers to guess passwords.<br>**Threat:** Elevation of Privilege | 5 / 5 / **25** | Only passwords are used. There are no rules for long or complex passwords. | **SR 1.1 RE 2/3:** Use MFA for all logins to the system, especially from outside the network. | 2 / 3 / **6** |
| **Firewalls** | The firewalls are open. This means a hacker can move easily from the IT network to the turbines. **Threat:** Tampering | 4 / 5 / **20** | Firewalls are installed but they do not block any traffic right now. | **SR 5.1 & 5.2:** Use firewalls to separate the network into zones. Block all traffic that is not needed. | 2 / 3 / **6** |
| **Software Updates** | Systems are not updated regularly. This leaves "open doors" for hackers. <br> **Threat:** Tampering | 5 / 5 / **25** | We only fix problems after they happen. There is no plan for updates. | **SR 6.1 & 7.8:** Keep a list of all devices and check them for security problems regularly. | 2 / 2 / **4** |
| **Network Monitoring** | We cannot see what is happening in the network. We might not notice an attack. <br> **Threat:** Information Disclosure | 5 / 5 / **25** | There is no system to watch the traffic or alert us about problems. | **SR 6.2:** Install a system (IDS) that monitors the network for strange activity. | 2 / 3 / **6** |
